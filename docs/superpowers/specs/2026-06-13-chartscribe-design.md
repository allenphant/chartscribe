# ChartScribe — AMR 圖表描述產生器　設計規格

> 日期：2026-06-13
> 狀態：設計已確認，待撰寫實作計畫
> 相關文件：`報告結構分析.md`（領域知識與寫作守則來源）

---

## 1. 目的

一個**純前端、零伺服器**的網頁工具，讓使用者放入圖表（圖片），透過 Gemini 多模態 API 產生**客觀描述性文字**，並以純文字 / Markdown / LaTeX 三種格式輸出供複製或下載。

內建「臺灣抗生素抗藥性監視年報」領域知識作為風格預設之一，能產出貼合年報原文風格的描述；同時保留通用模式可描述任何圖表。

## 2. 範圍與非目標

**範圍內**
- 批次上傳多張圖表，逐張產生描述。
- 全域風格預設 + 全域共用指示 + 每張卡片專屬說明欄。
- 三種輸出格式（純文字 / Markdown / LaTeX），個別卡片可複製/下載，並可「合併全部」成單一文件。
- 使用者自帶 Gemini API key，存於瀏覽器 localStorage。

**非目標（YAGNI）**
- 不寫後端伺服器、不代理 API。
- 不做帳號系統、不做雲端儲存。
- 不支援本地模型（Gemma/Ollama）——本版只接 Gemini 雲端 API。
- 不做圖表「重繪」或數據抽取成結構化表格——只產生描述文字。

## 3. 架構決定

| 項目 | 決定 |
|------|------|
| 部署模型 | 純前端 + 雲端 Gemini API（使用者自帶 key） |
| 實作方式 | 無建置・原生 ES module（可雙擊開啟或丟靜態空間） |
| 領域範圍 | 混合：通用為底 + AMR 風格預設 |
| 輸入 | 批次多圖 + 全域共用指示 + 每卡片專屬說明欄 |
| 輸出組織 | 個別區塊 + 合併鈕 |
| 預設模型 | `gemini-2.5-flash`（設定面板可改） |

## 4. 使用者流程

```
輸入 Gemini API key（存 localStorage）
        ↓
拖拉 / 貼上(Ctrl+V) / 選取 多張圖表 → 每張成為一張卡片
        ↓
全域：風格預設 + 共用指示 + 輸出格式
每卡片：選填專屬說明欄
        ↓
按「產生全部」→ 逐張依序呼叫 Gemini Vision（各自狀態）
        ↓
每卡片顯示輸出，可切換格式、複製、下載、重生
        ↓
「合併全部」→ 整批串成單一文件，複製或下載
```

## 6. 狀態模型

每張卡片：
```js
{
  id: string,
  file: File,
  thumbnailURL: string,     // object URL
  perCardContext: string,   // 選填，該圖專屬數據/指示
  status: 'queued' | 'processing' | 'done' | 'error',
  markdown: string,         // AI 回傳的正規形式，唯一真實來源
  errorMessage: string
}
```

全域狀態：
```js
{
  apiKey: string,           // localStorage
  model: string,            // localStorage, 預設 'gemini-2.5-flash'
  stylePreset: string,      // 預設 key
  sharedInstructions: string,
  outputFormat: 'plain' | 'markdown' | 'latex',
  cards: Card[]
}
```

## 7. Gemini 串接

- 圖片以 `FileReader` 讀成 base64，透過 `inlineData`（`mimeType` + `data`）內嵌送出。支援 PNG / JPG / WebP。
- 端點：`https://generativelanguage.googleapis.com/v1beta/models/{model}:generateContent?key={apiKey}`（支援瀏覽器 CORS）。
- 請求內容 = 系統指示（風格預設 prompt + 客觀守則 + 語言規則） + 共用指示 + 該卡片專屬說明 + 圖片。
- 批次採**依序**呼叫（for-await 串行），逐張更新狀態；單張失敗不中斷其餘。
- 429（速率限制）：自動稍候重試一次；仍失敗則標 error。

## 8. 語言與用詞規則（重要）

注入所有 prompt 的硬性規則：
1. **輸出語言預設繁體中文**；若共用指示要求其他語言則從之。
2. **生物醫學專有名詞一律保留英文**：菌種學名（如 *Escherichia coli*、*K. pneumoniae*）、抗生素名（如 vancomycin、meropenem）、抗藥性縮寫（CRAB、MRSA 等）不翻譯。
3. **菌種學名以斜體呈現**（Markdown 用 `*...*`）。
4. **客觀守則**（取自 `報告結構分析.md` §5）：
   - 數據說什麼就只寫什麼，不加臨床詮釋（如「可作為可靠治療選擇」）。
   - 敏感率（% susceptible）與抗藥率（% resistant）為互補概念，不可混用。
   - 分母 < 20 之數據不呈現/不強加。

## 9. 風格預設

| 預設 key | 名稱 | 注入的圖表描述邏輯 |
|----------|------|------------------|
| `general` | 通用 | 一般客觀圖表描述 |
| `amr_susceptibility` | AMR－敏感性圖譜 | 逐抗生素類別描述敏感率，純客觀 |
| `amr_resistance_bar` | AMR－抗藥性百分比橫條圖 | 描述抗藥率、95% CI、CO/HO 差異 |
| `amr_composition` | AMR－菌株組成堆疊長條圖 | 前三名占比、多年升降方向 |
| `amr_trend` | AMR－多重抗藥趨勢折線圖 | 108→113 走勢、醫院層級高低差異 |

所有 `amr_*` 預設皆夾帶 §8 的客觀守則。`general` 套用語言規則但不夾帶 AMR 特定守則。

## 10. 三種輸出格式產生方式

**單一真實來源策略**：AI 只回傳一份 **Markdown**，客戶端確定性轉換：

| 格式 | 產生方式 |
|------|---------|
| Markdown | AI 原始輸出原樣 |
| 純文字 | 去除 Markdown 標記（標題符號、`*`、清單符號等），保留段落 |
| LaTeX | 標題→`\section`/`\subsection`；`**粗體**`→`\textbf{}`；`*斜體*`→`\textit{}`；清單→`itemize`；段落保留 |

理由：比叫 AI 一次吐三種格式更省 token、三版本保證一致。轉換為純函式，易測。

**合併全部**：把所有 `done` 卡片的當前格式輸出串接，各段以分隔（如圖片檔名標題）區隔，提供複製 + 下載（`.txt` / `.md` / `.tex`）。

## 11. 錯誤處理

| 情境 | 處理 |
|------|------|
| 無 key / key 無效 | 卡片顯示明確訊息，引導至設定面板 |
| 單張 API 失敗 | 該卡片標 error + 「重生」鈕，不中斷整批 |
| 非圖片檔拖入 | 拒收並提示 |
| 429 速率限制 | 自動稍候重試一次，仍失敗標 error |

## 12. 檔案結構

```
index.html        介面骨架
styles.css        樣式（CSS 變數，無框架）
js/
  app.js          主控 + 全域狀態 + 事件綁定
  ui.js           渲染卡片與輸出區
  gemini.js       API 呼叫（薄封裝）
  formats.js      md→純文字 / md→LaTeX（純函式）
  presets.js      風格預設定義 + prompt 組裝
  storage.js      localStorage 存取 key/設定
```

每個模組單一職責，可獨立理解與測試。

## 13. 測試策略

- 純函式以 Node 內建 `node:test` 測試（零額外依賴、無建置）：
  - `formats.js`：md→純文字、md→LaTeX 各種標記轉換正確性（含菌種斜體 `*E. coli*` → `\textit{E. coli}`）。
  - `presets.js`：prompt 組裝是否正確夾帶語言規則、客觀守則、各預設片段。
- Gemini 網路呼叫保持薄、不測副作用。
- 手動驗收：用本年報實際圖表（如圖 3-1-1、圖 3-2-1）跑一遍，比對輸出是否貼合年報風格。

## 14. 驗收標準

1. 可在無建置情況下開啟 `index.html` 直接使用。
2. 輸入 key 後，拖入一張 AMR 圖表 + 選對應風格預設 → 產出繁中、專有名詞保留英文、菌種斜體、不含臨床詮釋的描述。
3. 三種格式可正確切換、複製、下載。
4. 多張圖批次處理，單張失敗可獨立重生。
5. 「合併全部」可下載單一文件。
6. `node:test` 純函式測試全數通過。
